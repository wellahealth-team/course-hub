Resolve merge conflicts (why they happen & how to fix)

Primary doc: About merge conflicts

Do‚Äëit (script a safe, tiny conflict)

# In main:
git switch main && git pull --ff-only
echo "title=original" > config.ini
git add config.ini && git commit -m "base: add config title"
git push

# Branch A:
git switch -c feature/title-a
sed -i 's/original/A-title/' config.ini
git commit -am "A: change title"
git push -u origin feature/title-a

# Branch B (from main):
git switch main && git pull --ff-only
git switch -c feature/title-b
sed -i 's/original/B-title/' config.ini
git commit -am "B: change title"
git push -u origin feature/title-b

Open PRs for both branches into main. Merge one; the second will conflict. Use the UI or your editor to resolve, commit, and complete the merge. Then append a proof line and push:

Prove resolution

echo "CONFLICT_RESOLVED: <your-github-username>" >> TAGLINE.txt
git add TAGLINE.txt
git commit -m "fix: resolve merge conflict in TAGLINE.txt"
git push
5

Undo toolkit (pick the safe option)

Overview: Undoing changes. Deeper docs: restore, revert, reset, reflog.

Do‚Äëit (micro‚Äëexercises)

# A) Unstage a file without losing changes
git add app.js
git restore --staged app.js

# B) Revert a single bad commit (safe on shared branches)
git log --oneline          # copy the SHA
git revert <sha>           # creates a new commit that undoes it

# C) Reset last commit locally (unpushed) but keep changes staged
git reset --soft HEAD~1

# D) Recover with reflog (find where HEAD was moments ago)
git reflog                 # copy the earlier HEAD
git reset --hard <sha>     # bring your tree back
