MDN Docs

Focus: using required, pattern, minlength, maxlength, and ensuring all controls are accessible to assistive technologies.

Do‚Äëit (follow along)

Tip: Test your forms by trying to submit without filling required fields. The browser should prevent submission and show helpful error messages.

Troubleshooting

Labels not working

  • Ensure the for attribute on the label exactly matches the id on the input.
  • Place the for attribute on <label>, not on the input itself.
  • Each id must be unique on the page.

Radio buttons allow multiple selections

  • All radio buttons in a group must share the same name attribute.
  • Each radio button needs a unique id for its label.
  • Only one radio button with the same name can be selected at a time.

Form validation not working

  • Ensure your button has type="submit" (default for buttons inside forms).
  • Check that required attributes are on the correct inputs.
  • Use appropriate input types (e.g., type="email") for automatic validation.

Accessibility issues

  • Use DevTools ‚Üí Accessibility pane to verify label associations.
  • Test keyboard navigation (Tab, Space, Enter) through your entire form.
  • Add aria-describedby for help text that explains validation rules.
